Цель: Создание интерфейса,который создает объект. При этом,выбор того, экземпляр какого класса создавать
остается за классами,которые имплементируют данный интерфейс.

Используется для делегирования создания экземпляров ,другому классу.

Пример использования:
- Заранее неизвестно,экземпляры,какого класса нужно будет создавать.
- Класс спроектирован таким образом,что создаваемые им объекты имеют свойства определенного класса.

Есть интерфейс.Создает класссы,реализующие этот интерфейс.Затем создаем класс factoty.
Создаем enum.В класса factory есть метод create  параметром String.Далее по параметру
ищем в enum константу,и в зависимости от этой константы с помощью switch присваевам
полю значение.
